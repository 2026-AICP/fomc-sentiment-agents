> 담당: EconPilot · 상태: 정합성 분석 골조 완료(방식 ①), 서프라이즈 방식(②)·VIX·이벤트 스터디는 후속 과제
> 대상 데이터: 회의 6건 (2000년 5건 + 2025년 1건), 진짜 FinBERT(finbert-cal) 기준

## 1. 무엇을 했나 (구현 산출물)

| 파일 | 역할 |
|---|---|
| `analysis/collect_market.py` | meetings 날짜 기준 ±7일 S&P500·VIX 시장데이터 수집 → market 테이블 적재 |

## 2. 핵심 설계 결정 (왜 이렇게 했나)

1. **수익률 단위 = % (RETURN_AS_PCT=True)**
   - `spx_ret_cc`는 소수(0.006)가 아니라 %(0.61) 단위로 저장. 팀 전체 동일 규약 필요.
   - 통계 처리 시 스케일만 다를 뿐 상관·회귀 결론은 동일하나, 표시·해석 일관성을 위해 % 채택.

2. **톤 집계방식 = conf_weighted 기본**
   - FOMC 성명문은 보일러플레이트 비중이 커 단순평균(label_avg)은 신호가 희석됨.
   - 확신도 가중이 핵심문장 신호를 더 잘 반영.

3. **시장 반응 시점 = 발표 + 1거래일 (REACTION_OFFSET=1)**
   - FOMC 발표는 미 동부 오후 2시 → 당일 종가엔 반응이 섞임. 다음 거래일 종가-대-종가 수익률로 반응 측정.

4. **divergence 정의 = 방향 일치 방식 (방식 ①)**
   - `alignment_flag`: 톤 부호 == 반응 부호 → 1, 아니면 0.
   - `divergence_score`: 부호 반대일 때 |톤|×|반응|, 같으면 0.
   - 서프라이즈 기반(방식 ②)은 자리만 비워둠 (compute_alignment()만 교체하면 승격 가능).

## 4. 결과 해석 — 50%는 실패가 아니다

- 겉보기엔 "동전 던지기"지만, 괴리 사례가 무작위가 아니라 **시장 선반영 메커니즘**을 반영.
- 2000-02/03: 톤은 매파(악재)인데 시장 상승 → 긴축이 이미 예상돼 있어 "예상대로"에 안도.
- 2000-05: 톤 최강 매파 + 시장 하락 → 실제 50bp라는 **서프라이즈**여서 방향대로 반응.
- **함의**: 시장은 톤 자체가 아니라 "예상 대비 서프라이즈"에 반응. → 방식 ②(서프라이즈 기반)로 승격이 타당함을 데이터가 시사.

## 5. 다음 할 일 (인수인계)

1. **서프라이즈 기반(방식 ②) 승격** — 직전 회의 대비 톤 변화 vs 실제 반응의 잔차로 divergence 재정의. 연속된 2000년 5건으로 계산 조건은 이미 충족. `compute_alignment()` 교체만으로 가능.
2. **회의 표본 확대** — 드롭박스 `FOMC Statements`에 2000~2021년 원문 존재. fixtures에 넣고 pipeline 재실행하면 자동 처리(순회 코드 완비). 통계적 유의성 위해 최소 수십 건 권장.
3. **VIX 연동 분석(문서 28번)** — vix_chg 이미 수집됨. 불확실 톤 vs VIX 관계·선행/후행.
4. **이벤트 스터디(문서 30번)** — 발표 전후 ±N일 비정상 수익률. 기준선 설정에 표본 다수 필요.
5. **[스키마 검토] meetings에 model_tag 추가** — 현재 sentences엔 model_tag가 있으나 meetings엔 없어, 더미↔진짜 톤이 덮어써짐. 엔진별 비교 보존하려면 추가 필요(팀 합의 사안).

## 6. 알려진 한계 / 주의

- 회의 6건은 표본이 작아 정합률·상관은 참고용. 결론 도출용 아님.
- 2000년 성명문 일부에 변환 노이즈(예: `thefederal` 공백 누락). 대량 확장 시 정제 고려.
- 국채금리(ust2y·ust10y)는 FRED 키 필요로 아직 NULL. 스키마 자리는 마련됨.
- FinBERT는 금융 감성 도구이지 hawk/dove 정책축 도구가 아님 → 방향은 맞아도 강도 해석은 신중히.

## 7. 재현 방법 (넘겨받는 사람용)

이 문서의 결과를 그대로 재현하려면, 아래 순서를 정확히 따른다.

```bash
# 0. 가상환경 활성화 + 의존성 (최초 1회)
source venv/bin/activate
pip install -r requirements.txt        # transformers, torch, yfinance 포함

# 1. 모델 배치 확인 (git에 없음 — 드롭박스에서 받아 배치)
#    models/finbert-finetuned/ 에 7개 파일 (pytorch_model.bin 418.75MB 등)
ls models/finbert-finetuned/

# 2. 원문 확인 — tests/fixtures/ 에 FOMC_<날짜>_statement.txt 형식으로 존재
ls tests/fixtures/

# 3. 진짜 FinBERT로 전 회의 감성분석 (fixtures 자동 순회)
SENTIMENT_ENGINE=finbert python3 pipeline.py

# 4. 시장 데이터 수집 (meetings 날짜 기준 ±7일)
python3 analysis/collect_market.py

# 5. 정합성 분석 (alignment_flag·divergence_score)
python3 analysis/analyze_alignment.py
```

- 회의를 추가하려면: 드롭박스 원문을 `FOMC_<날짜>_statement.txt` 형식으로 fixtures에 넣고 3~5단계 재실행. (순회 코드가 자동 처리)
- 3단계를 `SENTIMENT_ENGINE` 없이(더미로) 돌리면 안 됨 — 아래 8절 경고 참조.

## 8. ⚠️ 중요 경고 — meetings 톤 오염 주의

- **문제**: `sentences` 테이블엔 `model_tag`(dummy / finbert-cal)가 있어 엔진별로 값이 공존하지만, **`meetings` 테이블엔 `model_tag`가 없다.** 집계 톤은 `(date, method)`를 키로 INSERT OR REPLACE 되므로, **더미로 pipeline을 재실행하면 진짜 FinBERT 톤이 조용히 덮어써진다.**
- **결과**: 이 문서의 모든 톤·정합성 수치는 진짜 FinBERT(finbert-cal) 기준. 더미로 덮이면 분석 결론이 뒤집힘(예: 2025-01-29 톤이 진짜 +0.15 → 더미 -0.29로 부호 반전).
- **당장의 방어**: 진짜 결과 상태를 `data/fomc_backup_finbert_6meetings.db`로 백업해둠. 오염 시 복구:
  ```bash
  cp data/fomc_backup_finbert_6meetings.db data/fomc.db
  ```
- **근본 해결(후속 과제)**: `meetings`에도 `model_tag` 컬럼 추가 → 엔진별 톤 병렬 보관. 스키마 변경이라 팀 합의 필요.

## 9. Phase 5 / Phase 6 경계 (분업 명시)

Phase 5에서 **의도적으로 하지 않은** 것들 — 미완성이 아니라 범위 밖 / 후속:

| 항목 | 왜 여기서 안 했나 | 어디 소속 |
|---|---|---|
| 국채금리(ust2y·ust10y) 수집 | FRED API 키 필요, 배관 검증 우선 | Phase 5 후속(키 발급 후) |
| divergence "경고 후보" 임계값 판정 | 위험·기회 신호 생성 로직 | Phase 6 |
| lead-lag 상관, 이벤트 스터디 | 회의 표본 다수 필요 | Phase 5 후속(표본 확대 후) |
| 서프라이즈 기반 지표(방식 ②) | 방식 ①로 골조 먼저 (데이터가 승격 타당성 시사) | Phase 5 후속(다음 스텝) |

**넘겨받는 사람의 권장 첫 스텝**: 회의 표본을 늘린 뒤(드롭박스 원문 활용) → 서프라이즈 기반(방식 ②)으로 divergence 재정의. `compute_alignment()` 교체만으로 가능하도록 설계돼 있음.
