# Phase 4 — 인덱스 집계 방식 비교 + 보정 효과

> 보정 엔진(finbert-cal, T=3.1)으로 2000~2025 성명문 214회의 인덱스 재계산 후 비교.

## 1. 작년 방식 vs 새 방식 (Phase 4 합격선)

| | label_avg (작년) | conf_weighted (새 방식) |
|---|---|---|
| 정의 | argmax 라벨(-1/0/1) 평균 | score를 확신도(1−정규화 엔트로피)로 가중 |
| 표준편차 | 0.251 | 0.157 |
| 상관 | — | +0.944 (대체로 같이 움직임) |

**핵심 발견**: label_avg가 거의 0(긍·부정 라벨 상쇄)인 회의 **19건 중 11건에서, conf_weighted는 0이 아닌 신호를 잡았다.**

→ 작년 방식은 "긍정 문장 2개, 부정 문장 2개"면 무조건 0으로 뭉개지만, 확신도 가중은 **"모델이 어느 쪽을 더 확신했나"**를 반영해 미세 신호를 살린다. 이것이 작년 대비 인덱스 개선점이다.

## 2. 보정 효과 (baseline vs calibrated)

| | baseline (T=1, 과신) | calibrated (T=3.1, 정직) |
|---|---|---|
| 표준편차 | 0.245 | 0.157 |
| 범위 | [−0.69, +0.59] | [−0.43, +0.45] |
| 상관 | — | +0.978 (패턴 유지) |

→ 보정하면 인덱스의 **극단 스윙이 완화**된다(2007 신용위기, 2020 COVID 스파이크 등). 패턴은 그대로(상관 0.978) 유지하면서 **과신에서 오는 과장된 변동만 줄어** 더 정직한 인덱스가 된다.

## 3. 결론

- 새 집계(conf_weighted)는 작년 방식이 놓치는 신호를 잡는다(19건 중 11건).
- 캘리브레이션은 인덱스를 더 정직하게(과장 스윙 완화) 만든다.
- 두 개선 모두 작년 대비 인덱스 신뢰도를 높이는 방향이다.
