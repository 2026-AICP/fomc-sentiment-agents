# FinBERT 온도보정(T=3.1) 방어 — reliability diagram + 엔트로피 분포

> 확신도 가중(conf_weighted) 지수가 정직하려면, 모델 확신도가 실제 정확도와 맞아야 한다.
> 온도 스케일링(T=3.1)이 과신을 교정함을 라벨 데이터로 시각·정량 입증.
> 코드: `analysis/reliability_diagram.py` · 그림: `docs/figures/calibration/`

## 1. 방법 (정직한 계산)

- **라벨**: 사람 라벨 성명문 문장 150개(`labeling_ground_truth_150.csv`) 중 DB 텍스트 매칭 **143개**
- **모델 통과 → raw logits → T=1·T=3.1 softmax 직접 산출** (양쪽 일관, 한 번의 추론)
- 확신도 = max(prob), 정확도 = (argmax == 사람 라벨). ECE = 확신도 구간별 |정확도−확신도| 가중평균

## 2. 결과

| 지표 | raw (T=1) | 보정 (T=3.1) |
|---|---|---|
| **ECE** (0=정직) | **0.294** | **0.112** |
| 확신도 평균 위치 | ~0.99 (과신) | 대각선 근접 |
| **엔트로피 평균** | **0.24** (거짓 확신) | **0.79** (정직, 최대 1.10) |
| 정확도(argmax) | 63.6% | 63.6% (라벨 불변) |

## 3. 해석 (포스터 문장)

**① reliability diagram** — raw 모델은 **확신 99%인데 실제 정확도 67%** (대각선 한참 아래 = 과신).
T=3.1 보정 후 곡선이 **대각선에 근접**해 "확신도 = 실제 맞을 확률"이 정직해짐 (ECE 0.294 → 0.112, **62% 감소**).

**② 엔트로피 분포** — raw는 대부분 엔트로피 ≈ 0 (모든 예측이 "확실"하다고 우김).
보정 후 **honest한 불확실성**으로 퍼짐 (평균 0.24 → 0.79). 이 엔트로피가 conf_weighted의 가중치라,
**보정 없이는 불확실한 문장도 과대 가중**되어 지수가 왜곡됨 → **T=3.1이 지수 정직성의 전제**.

## 4. 한계 · 다음

- **baseline은 성명문 143문장**. presser는 다른 텍스트(구어체 Q&A) → **presser 라벨 완성 후** 같은 스크립트로 presser reliability 추가 예정 (`data/labeling/` 진행 중).
- ECE 0.112는 이 143개 in-sample 값. 문서화된 2-fold CV(0.07~0.10)와 같은 개선 방향.
- 재현: `python3 analysis/reliability_diagram.py` (라벨셋 교체 시 인자로).
