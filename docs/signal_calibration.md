# 신호 임계값(θ) 보정 · 민감도 분석

> 데이터 기반 θ 설정 + 민감도 분석으로 "자의성" 논란을 차단하는 근거 자료.
> (2026-07-03, 59개 회의 · 코드: `analysis/signals.py` `Thresholds`)

## 1. 왜 — "근거"의 3층위

```
① 신호가 발동한다 (θ 적용)        ← 근거 아님. 규칙 적용일 뿐
② θ가 자의적이지 않다 (데이터 기반) ← "체리피킹 아님" 방어
③ 신호가 실제로 의미있다 (검증)     ← ★ 진짜 근거 (상관·백테스트)
```

- 표본이 작을 때(59개) θ를 "결과 좋게" 최적화하면 **과최적화(overfitting)** → 오히려 약점.
- 따라서 **원칙 기반 θ(분포 분위수) + 민감도 분석**이 정답. "완벽한 θ 하나"보다 "θ에 안 흔들림"이 강한 근거.

## 4. 민감도 분석 ⭐ (핵심 근거)

θ 배율별 '괴리' 신호가 뜬 회의:

| θ 배율 | 괴리 | 회의 |
|---|---|---|
| ×0.5 | 16 | 2000·2021·2022(6)·2023(3)·2024·2025 |
| ×1.0 | 8 | 2000-02·2000-06 · 2021-12 · 2022-05·11·12 · 2023-09 · 2024-07 |
| ×1.5 | 2 | 2023-09-20 · 2024-07-31 |
| ×2.0 | 0 | — |

→ **θ를 어떻게 잡든 괴리는 일관되게 위기·긴축기에 집중** (2000 닷컴 · 2022-23 급속 인상기 · 2024 전환점).
→ 가장 강한 괴리(2023-09·2024-07)는 **θ를 2배로 올려도 유지**.
→ 즉 **"괴리 = 스트레스 시점" 결론은 θ에 흔들리지 않음** = 자의성 논란 차단.
(개수는 θ에 따라 변하지만, 성격·집중 시점은 불변.)
